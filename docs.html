Documentation

Welcome to the official Alphonso Ecosystem documentation. Alphonso is a local-first AI companion desktop application designed for maximum privacy, efficiency, and multi-agent collaboration.

Local-First Philosophy: Alphonso runs entirely on your machine. All data processing, memory storage, and agent orchestration happen locally, ensuring your data never leaves your device without explicit approval.

Installation

Alphonso is built with Tauri v2 and runs on Windows, macOS, and Linux. Follow these steps to get started:

# Clone the repository
git clone https://github.com/Thatisshayan/AlphonsoEcosystem

# Install dependencies
npm install

# Run in development mode
npm run tauri dev

Core Stack

Layer Technology
Frontend React 18, Vite 5, Tailwind CSS
Backend Rust 1.77, Tauri 2.11
Local AI Ollama (llama3.2:3b default)
Storage SQLite (rusqlite), WAL mode

Agent Orchestration

The ecosystem is powered by a 9-agent system coordinated through the Jose orchestration pipeline. Each agent has a specific contract enforced by the agentContractService.

// Example Orchestration Flow
User Input → Jose Intake
  → Task Decomposition
  → Agent Assignment (Contract Check)
  → Execution (Packet Engine)
  → Maria Governance Audit
  → Final Merge & Confirm

Security & Privacy

Privacy isn't a feature; it's the foundation. Alphonso employs a multi-layered security model:

  • Fail-Closed Policy: If an action is ambiguous, it is blocked by default.
  • Approval Gates: Any external data send requires explicit user confirmation.
  • Encrypted Storage: All local data is stored in a secure SQLite instance.
  • Zero-Cloud Dependency: Core operations work 100% offline.
